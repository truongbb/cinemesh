### Goal
Fix the Request ID tracing by consolidating redundant Feign interceptors and ensuring the Spring Cloud Gateway returns the generated ID to the frontend.

### Assumptions
- The system uses standard synchronous Feign clients (Servlet context is available, or MDC is properly copied if using minimal async).
- `CorrelationIdGlobalFilter` in the Gateway is currently dropping the ID from the response.
- `FeignCorrelationInterceptor` and `FeignClientConfig` are conflicting because both define a `RequestInterceptor`.

### Plan

1. **Update Gateway to return Request ID in Response**
   - Files: `infrastructure/cinemesh-gateway/src/main/java/com/cinemesh/cinemeshgateway/filter/CorrelationIdGlobalFilter.java`
   - Change:
     - Mutate the response headers to include `X-Request-Id` so the frontend receives the same ID generated by the Gateway.
   - Verify: `curl -v http://localhost:<gateway-port>/...` should show `< X-Request-Id: <uuid>` in the response.

2. **Consolidate Feign Interceptors**
   - Files:
     - `common/src/main/java/com/cinemesh/common/interceptor/FeignCorrelationInterceptor.java`
     - `common/src/main/java/com/cinemesh/common/config/FeignClientConfig.java`
   - Change:
     - Move the `Authorization` header propagation logic from `FeignClientConfig` into `FeignCorrelationInterceptor`.
     - Remove the `@Bean RequestInterceptor` from `FeignClientConfig` entirely to prevent two interceptors from running and potentially clashing.
   - Verify: Run `mvn clean compile` in `common/` to ensure syntax is correct.

3. **Verify MDC Fallback logic in Feign Interceptor**
   - Files: `common/src/main/java/com/cinemesh/common/interceptor/FeignCorrelationInterceptor.java`
   - Change:
     - Ensure the consolidated interceptor checks `MDC` first for `requestId`, falls back to `RequestContextHolder`, and injects *both* `X-Request-Id` and `Authorization` into the Feign `RequestTemplate`.
   - Verify: Ensure the microservices can compile and boot without interceptor bean conflicts.

### Risks & mitigations
- **Risk**: Async Feign calls losing both MDC and `RequestContextHolder` thread context.
- **Mitigation**: If services are entirely synchronous (default Spring Boot), this works. If they use `@Async`, a custom `TaskDecorator` will be needed later. For now, consolidating the interceptors fixes the immediate conflict.

### Rollback plan
- Revert changes to `CorrelationIdGlobalFilter`, `FeignCorrelationInterceptor`, and `FeignClientConfig` using `git restore`.
